# 上海区域项目销售及售前会议纪要（提炼汇总）

## 一、工作机制和方法

- **信息同步机制**：需建立统一 issue 表、项目一页纸、索引页子，同步至研发 / 前场，避免信息断层；禁止口头传递，所有问题书面记录闭环。
- **需求挖掘方法**：深入客户实战场景，挖透需求本质（如反诈需区分「路过」与「进入 ATM」），避免仅停留在表面需求。
- **优先级管理**：项目按紧急 / 重要分级，重点区域优先推进；上海区域项目密集，需排期分批升级，避免集中攻坚导致资源不足。
- **复盘闭环机制**：对系统效果、数据准确性、客户反馈定期复盘，高频问题专项整改；业务流、算法逻辑、预警规则需全链路闭环验证。
- **跨部门协作**：售前、研发、前场、商务联动，售前输出方案，研发负责落地，前场反馈现场问题，商务对接预算。

## 三、销售策略

- **重点客户聚焦**：优先服务黄浦（实战型、易出成果）、松江（老客户、客情稳定）、青浦（进博会节点、政策倾斜）；闵行重点维护分管治安 / 科信 / 交通的副局长。
- **价值包装策略**：主打实时拒档、L1–L5 打通、大模型实战成效；将「摄像头质量评估」「离线语义搜索」包装为差异化亮点。
- **商务谈判原则**：先确认预算再落地定制需求，避免无成本开发；上海客户成本敏感，主打「高性价比、迭代升级、长期价值」。
- **竞品应对策略**：针对大华（青浦主场、大模型强势），突出我方实战战果、实时拒档时效、L1–L5 打通优势；避免正面硬刚，聚焦差异化场景突破。
- **试点带动策略**：黄浦、青浦打造标杆案例，辐射全市；重点区域（虹桥火车站、南京东路）优先落地核心功能，形成示范效应。

## 四、市局考核项

- **大模型考核**：OCR 识别（横幅 / 涂鸦文字区分正负面内容）、以图搜图、涉恐涉政识别、人员聚集预警；我方排名靠后，核心问题是功能缺失、响应慢、误报高。
- **实时拒档考核**：时效（目标 1 分钟，当前黄浦 3 分钟）、轨迹追踪准确率、精神病人 / 流浪人员预警闭环率。
- **数据考核**：摄像头建模成功率、抓拍数量、解析率、系统稳定性、上云适配度。
- **算法能力考核**：非机动车违规识别、夜间违停预警、人群异常行为识别、跨时空追踪能力。

## 五、关键技术参数定位

**2 亿人脸、4 亿人体**：规模极大，远超苏州（1.1 亿人脸），属于上海顶级数据体量，直接导致系统压力大、硬件成本高、升级周期长，需分优先级（重点区域实时、非重点区域准实时）处理。

## 六、功能的用法

| 能力 | 对接与场景 |
| --- | --- |
| **图搜** | 对接第三方平台（麒麟神智）、满足短期（1 个月内）查询、普通检索需求、非核心高频场景。 |
| **档搜** | 对接情报 / 数据部门、支持长周期（1 年）检索、核心实战场景、高频查询、L4/L5 打通核心能力。 |

## 七、试用系统的规划原则和方法

- **优先级原则**：重点区域（人流密集、安保核心）优先部署实时功能；非重点区域部署准实时功能，降低硬件压力。
- **版本迭代原则**：先稳定 3.0 基础版，再逐步叠加大模型、L1–L5 打通等高级功能；避免一次性全量升级导致故障。
- **硬件适配原则**：高人流区域（黄浦南京东路）增加硬件配置；普通区域标准配置，平衡成本与性能。
- **试点先行原则**：单区域小范围试点核心功能，验证效果后再全量推广；黄浦、青浦优先试点 L1–L5 打通。

## 八、客情、友商动态

### （一）客情

- **黄浦**：史芝（业务型领导）认可我方，实战意愿强，易出成果，重视 L1–L5 打通；客情最优。
- **松江**：老客户，客情稳定，一把手调任闵行，新领导未到岗，决策暂缓；重视系统稳定性和基础算法。
- **闵行**：多分管领导，需求分散，需重点维护；重视实时拒档和大模型考核。
- **青浦**：新局长胡局（71 年、经侦出身）激进、重视科信建设；进博会节点，需求迫切，易争取资源。
- **静安**：进度滞后，客户无强推进意愿，优先级较低。

### （二）友商动态

- **大华**：青浦主场，大模型强势，提供免费设备，强势介入治安总队模型试点；主打算法编排、多模态能力，挤压我方空间。
- **其他厂商**：市局考核中部分功能优于我方，响应速度快、误报率低；聚焦 OCR、以图搜图等考核项突破。

## 九、上海地域特点（中间库）

- **数据规模**：人脸 / 人体数据体量全国最大（2 亿人脸、4 亿人体），远超其他城市，系统架构、硬件适配、算法优化难度更高。
- **中间库依赖**：实名中间库集中在市局，分局无独立库，跨区数据同步、档案关联依赖市局审批，流程繁琐，影响功能落地（如静安实名库未同步）。
- **双网 / 多网架构**：公安网暂未全覆盖，多数区域为三网运行，硬件成本翻倍，升级周期拉长。
- **区域差异大**：黄浦（高人流、实战优先）、青浦（展会 / 安保优先）、松江（郊区、基础维稳）、闵行（综合管理），需求差异化明显，需定制化方案。

## 十一、补充维度：核心痛点与改进方向

- **核心痛点**：系统稳定性不足、大模型功能缺失、实时拒档时效不达标、跨部门信息同步断层、硬件成本高、中间库对接流程繁琐。
- **改进方向**：优先修复松江 1.6 版本 bug；加急开发 OCR、以图搜图功能；优化实时拒档算法，目标时效 1 分钟；建立跨部门信息同步机制；分区域适配硬件配置；推动市局简化中间库对接流程。
